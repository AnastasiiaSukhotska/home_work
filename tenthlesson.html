Команда Действие
git add добавляет содержимое рабочего проекта для последующего коммита, если поставить . - добавляет все файлы
git commit сохраняет изменения в файлах,добавленных с помощью git add
git status показывает состояния файлов: какие файлы изменены, но не добавлены через add и commit
git config может быть использована для указания пользовательских настроек, таких как электронная почта, имя пользователя, почта и т.д.
git init эта команда используется для создания GIT репозитория
git push позволяет поместить изменения в главную ветку удаленного хранилища
git checkout может быть использована для создания веток или переключения между ними. К примеру, следующий код создаст новую ветку и переключится на нее: git checkout -b <имя-ветки> Чтобы просто переключиться между ветками используйте: git checkout <имя-ветки>
git remote позволяет пользователю подключиться к удаленному репозиторию. Данная команда отобразит список удаленных репозиториев, настроенных в данный момент: Эта команда позволит пользователю подключить локальный репозиторий к удаленному серверу: git remote add origin <адрес.удаленного.сервера>
git branch может быть использована для отображения, создания или удаления веток. Для отображения всех существующих веток в репозитории введите: git branch
git merge используется для объединения ветки в активную ветвь. Применение: git merge <имя-ветки> Возможно возникновение конфликтов. После их решения обязательно провести git innit без комментария
git diff используется для выявления различий между ветками. Для выявления различий с базовыми файлами, используйте git diff --base <имя-файла> Следующая команда используется для просмотра различий между ветками, которые должны быть объединены, до их объединения: git diff <ветвь-источник> <ветвь-цель> Для простого отображения существующих различий, используйте: git diff
git log отобразит список всех коммитов в ветке вместе с соответствующими сведениями
git clone получение копии существующего Git-репозитория; клонирование репозитория осуществляется командой git clone (url )
git pull это git fetch+git merge;команда добавляет имеющиеся на удаленном репозитории изменения в локальный репозиторий(могут быть конфликты!)
git fetch загружает изменения из удаленного репозиория, но не обьеденяет их с локальным репозиторием
git revert дает возможность откатить комит не удаляя их, а создавая новый, который отменит изменения и доведет репозиторий до состояни того комита id которого указано
git reset если просто reset удаляет комиты до того комита, который мы задали, но не убирает изменения в файлеб что бы убрать изменения - git reset hard. использовать только до того как произвели push
